PART 1 — CONTEXT (get yourself fully caught up)

## The hackathon
This is the **Generative UI Hackathon Track** (CopilotKit + AG-UI). The submission
MUST visibly use: **CopilotKit + AG-UI** (live agent↔frontend loop), **A2UI**
(declarative generative UI surfaces), and **LinkUp** (source-grounded search).
Redis is optional. Judging: most creative use of Generative UI + A2UI; integration
of the three; technical difficulty; originality. Deliverables: public GitHub repo,
demo video, social post.

## The product — PeerReview.ai
An agentic, evidence-based **marking cockpit** for CS-assignment grading by TAs.
The UI is not a wrapper around the AI — the UI IS the product: the agent *generates*
the marking interface at runtime from the assignment, the repo, the real test results,
and the misconception detected, with a human approving every final mark.

Two phases, two LangGraph agents:
- **Phase 1 — Setup (`/setup`)**: understand assignment → LinkUp reference →
  generate tests → **validate them against a trusted reference solution** (only
  reference-passing tests may freeze) → **human approval gate** → freeze workspace.
- **Phase 2 — Review (`/review`)**: ingest submission (uploaded folder / public
  GitHub URL / seeded) → **run the frozen pytest suite FOR REAL** (subprocess +
  timeout) → detect misconception → **deterministic graph trace** (run reference
  BFS layers vs the student's actual returned path) → propose evidence-based scores →
  calibration memory → **human approval gate** (edit a score → feedback regenerates) →
  final feedback saved to a local archive.

## Architecture (adapted from CopilotKit's generative-ui-london-hackathon-starter)
- **Frontend** (`src/`): Next.js + CopilotKit. Pages `src/app/(pdf)/setup/page.tsx`
  and `.../review/page.tsx` (routes are `/setup`, `/review`; the `(pdf)` folder name
  is inherited — fine to rename if careful). Chat (left) + `SurfaceCanvas` (right).
  Runtime route `src/app/api/copilotkit-agents/route.ts` proxies to the agents on
  `:8123`. A2UI catalog: `src/a2ui/catalog/{definitions.ts, renderers.tsx}`.
- **Backend** (`agent/`): FastAPI + LangGraph on `:8123`, `main.py` registers
  `/setup` and `/review`. Domain logic in `agent/src/peerreview/`:
  `pipeline.py` (orchestration), `test_runner.py` (REAL pytest + graph trace),
  `analysis.py` (misconception detection + scoring + feedback),
  `linkup.py` (reference lookup + offline fallback), `ingest.py` (seeded/pasted/GitHub),
  `store.py` (local JSON: workspaces/calibration/feedback), `surfaces.py` (deterministic
  A2UI builders), `offline.py` (OFFLINE stub). Agents: `setup_agent.py`, `review_agent.py`.
  Model: Gemini via `langchain-google-genai`, default id `gemini-3.5-flash`.
- **A2UI catalog**: generic primitives (Stack/Card/Section/Badge/Callout/DataTable/
  Button/charts) reused heavily, plus custom panels: `VisualGraphTracePanel` (SVG —
  BFS layers vs student path), `GradeApprovalPanel` (interactive editable scores +
  approve), `TestResultsPanel`, `MisconceptionPanel`, `ReferenceContextPanel`, `CodeBlock`.
- **Fixtures**: `fixtures/peerreview/assignment-bfs/` (brief.md, rubric.json,
  reference_solution.py, tests_reference.py), `submission-dfs/`, `submission-bfs-correct/`,
  `calibration-history/` (5 seeded records).

## Key design decisions and WHY (do not regress these)
1. **Deterministic core, LLM only for prose.** Marks-bearing decisions (pass/fail,
   misconception label, proposed scores, graph trace) are deterministic and derived
   from REAL evidence. Gemini only orchestrates (which tool to call) and polishes
   prose (misconception explanation, feedback letter). This is what makes the grades
   trustworthy. Don't let "simplification" move scoring into the LLM.
2. **Generated tests are validated against a trusted reference solution** before they
   can be frozen — answers "how do you know the tests are correct?"
3. **Graph-search tests assert PATH PROPERTIES, not one exact path** (multiple shortest
   paths are valid). The DFS submission fails ONLY the minimal-length property test —
   that's the misconception signal.
4. **Honest sandbox scope.** The runner is subprocess + hard timeout + captured output,
   for CONTROLLED demo submissions only. Docker isolation is the documented hardening
   path. Do NOT claim a "secure sandbox."
5. **Reuse A2UI primitives; add few custom panels.** The agent composes surfaces from a
   trusted catalog via deterministic Python builders — reliable, handles variable-length
   data, no invalid-JSON risk.
6. **Human-in-the-loop via A2UI button actions.** Buttons dispatch named events
   (`approve_workspace`, `approve_grades`, …) → `SurfaceCanvas.onAction` → `agent.runAgent`
   → backend injects `log_a2ui_event`. No LangGraph interrupts needed.

## Current status (what's done + verified)
The full flow was verified end-to-end in a browser in **OFFLINE mode** (no keys):
setup → approve/freeze → review (DFS submission) → edit a score → approve → final
feedback saved. Real pytest ran (6/7 pass, minimal-length fails with the real
assertion), the DFS-vs-BFS misconception fired, and the SVG graph trace rendered
(green A→F→G vs orange-dashed A→B→C→D→E→G). `tsc --noEmit` is clean.

## Environment + how to run (THIS machine — important gotchas)
- pnpm installed via `npm i -g pnpm`; `uv` installed via `pip install uv` but NOT on
  PATH (use `python -m uv` or `~/AppData/Roaming/Python/Python314/Scripts/uv.exe`).
- Agent venv at `agent/.venv` (Python 3.12). If you must recreate it and `uv sync`
  errors on Python version, pin: `uv sync --python 3.12` or point at the concrete
  `...\uv\python\cpython-3.12.13-...\python.exe`.
- Run: `pnpm dev` (concurrently Next on :3000 + agent on :8123 via
  `scripts/run-agent.mjs`, which launches `agent/.venv` uvicorn directly — no uv at
  runtime). `.npmrc` has `verify-deps-before-run=false`; `pnpm-workspace.yaml`
  `allowBuilds` has esbuild/sharp=true. postinstall is a no-op.
- **CRITICAL uvicorn-restart gotcha**: `uvicorn --reload` runs a reloader PARENT +
  server CHILD. Killing only one orphans the other, which keeps serving STALE code on
  :8123 and makes new starts fail silently. If code edits don't show up in the browser
  even though disk is correct: `Get-Process python | Stop-Process -Force`, confirm
  `:8123` is free, then start exactly one agent. Verify the agent log shows
  "Uvicorn running on http://127.0.0.1:8123 ... Application startup complete".
- `.env` currently has `OFFLINE=1` and now a real `GEMINI_API_KEY` (the user added it).
  LinkUp key is NOT set (its panel falls back to a clearly-labelled bundled reference —
  that's expected; ignore LinkUp for now).
- Read `walkthrough.txt` and `README.md` in the repo root for the full narrative.

## Known minor issues to address
- First "Load/submit" click immediately after a hard page reload no-ops until CopilotKit
  connects (the send() guards on `agent` being ready). Harden this (disable controls
  until ready, or queue the action).
- The **live Gemini path has NOT been tested yet** (only OFFLINE). In particular,
  confirm `gemini-3.5-flash` is a valid model id for the user's AI Studio key; if not,
  set `MODEL` in `.env` to a model the key supports (e.g. a current `gemini-*-flash`)
  and confirm tool-calling + multi-turn work.
